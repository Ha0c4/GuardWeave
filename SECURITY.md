# Security Policy

## Supported Versions

GuardWeave is currently maintained on the latest `master` branch and the latest tagged release.

## Reporting a Vulnerability

Please do not disclose vulnerabilities, credential leaks, or bypass techniques in a public issue.

Send a private report to:

- `chenghao0628@outlook.com`

Include:

- A short description of the issue
- Impact and affected configuration
- Reproduction steps or a proof of concept
- Whether the issue affects hosted APIs, local models, or both

## What to Avoid in Reports

- Do not include real secrets or production credentials
- Do not attach full private system prompts unless necessary
- Do not post zero-day details in public issues before a fix is available
